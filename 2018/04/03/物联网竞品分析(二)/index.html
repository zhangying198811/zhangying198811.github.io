物联网企业参观及分析 ( 二 )

Apr 3, 2018 | 产品

1. 物联网平台市场背景

  • 1.1 物联网平台分类

  • 1） 提供连接性管理的物联网平台

    主要是针对终端(SIM卡)的通信通道提供连接性管理、诊断以及终端管理方面的功能。如思科的Jasper平台、爱立信的DCP、Telit的M2M平台、PTC的Thingworx和Axeda。

  • 2） 提供云服务为主的应用开发平台

    主要是提供设备与数据接入、存储和展现服务，尤其适合不需要关心后台服务系统运作细节的用户。如LogMeIn的xively、Yeelink、中国移动的OneNet、京东智能云、腾讯微信/QQ物联、阿里云、百度IOT、中兴通讯的AnyLink。

  • 3） 提供接入智能装置为主的应用开发平台

    和上一个类型的平台相当类似，但这种公司通常因为还在初创阶段，在云端的研发实力较弱，所以将重点放在智能装置的接入方面。比较典型的业者如Ayla Networks。

  • 4） 以大数据分析和机器学习为主的物联网平台

    如IBM的Bluemix和Watson、亚马逊AWS IoT、Microsoft的Azure。

  • 5） 行业云平台

    提供包括应用软件、基础架构、业务流程等完整服务。这种平台有些会专注在特定产业的垂直应用，如智能家居、智慧城市、智能农业等不同领域。这些物联网平台当然会有部分功能重迭或向彼此渗透发展的趋势。

  • 1.2 物联网数据分类

  • 1） 能源类数据

    与能耗相关的，或者计算能耗所需的相关数据，例如电流、电压、功率银子、频率等。获取能源数据，理解能源数据，分析并能达到节能的目的

  • 2）资产属性数据

    硬件资产数据，例如设备规格、参数等属性、设备的位置信息、设备从属关系。资产类数据主要用户资产管理，资产管理可以和ERP系统、MES系统、物流等系统对接。

  • 3）诊断类数据

    是指设备运行过程中检测设备运行状态的数据，诊断类数据可以有两类：一类为设备运行参数，例如设备输入/输出值，通常为传统工业自动化类数据；另一类为设备外围诊断数据，例如表面温度、设备噪声、设备震动等。外围诊断数据是预测性维护的重要元数据，也为深度控制模型提供依据。

  • 4）信号类数据

    是指信号类数据或者告警类数据。

2. 关于云栖大会

​ 阿里云正将云的能力无缝下沉，在全球布局边缘计算，实现万物智能。未来，数据中心将继续承担大量计算 任务，而一些实时决策和自主协作的过程将转移到边缘计算，形成中心+边缘的协同计算。阿里云已经完成城市、生活、制造、汽车四大物联网领域的核心技术布局。阿里云的AI提供了强大的视频识别、自然语言理解、声纹识别、人脸识别等AI能力，应用到工厂车间、养猪农舍等，大规模赋能实体经济。

3.关于智城云的思考

除非是拥有核心的专利，现在技术、产品都不是瓶颈，而且有什么创新的想法 一般会被大公司快速的复制或收购；初创企业10几 20个人做事情 和BAT企业100多个顶尖的人去运作同一件事情速度和资源上根本没办法比，我觉得就是这个时代的产品垄断。想突破垄断，对于个人或小团队而言，唯一的方法是缩小用户群体。就是我们应该专注于一个领域一项技术去解决一个问题。逐步积累资金、客户，然后进入大企业的行业再去提升技术宽度 深度 并扼杀有威胁的小企业。技术、产品已经不是壁垒 反而客户资源、信息、渠道是成败的关键。

· 3.1 PaaS**平台定位**

目前IOT成为阿里主赛道，但智城云如果是定位万物互联、生态链，其实和阿里是同一个赛道，对智城云是比较严峻的考验。

· 3.2 **智能家居行业定位**

目前只要是物联网平台一般都在做智能家居，一个是市场成熟度还不够高，另一个是进入的人太多。目前有四类厂家做智能家居：1）生产智能硬件，例如欧瑞博、南京物联、紫光，但目前一般靠贴牌生存。2）集成商，例如珠江智联，找第一类厂家贴牌并销售，要有足够的销售渠道 3）方案商，例如云智易，提供技术支持服务 4）方案集成商，将各硬件厂家设备打包提供。 智城云打算走的路线，目前市面上没有人在走。走第4条路线面临的问题有1）供应商压货周期 2）方案集成周期等 3）有没有地产商愿意买单 4）如何与前2种厂家PK

· 3.3 **新思路**

物联网软硬件无法独立存在，需紧密结合。1）成熟业务：模组+连接平台，重连接。重点发力边缘智能。据预测到2020年，将有超过500亿的终端与设备联网，未来超过50%的数据需要在网络边缘侧分析、处理和储存。模组+软件的收费模式，将云资源费用核算到软件中。2）垂直行业，软件系统+硬件设备。准备从济宁、青岛的制造业入手摸底（和李成计划中）
